title: MERGE INTO
description: Upsert and conditional delete on Iceberg tables.

<h2 id="overview">Overview</h2>
<p>`MERGE INTO` applies upsert and conditional delete logic to an Iceberg table using copy-on-write semantics. The target table must be managed by a registered `KrishivCatalog` Iceberg catalog.</p>
<Callout type="warn" title="Preview"> Copy-on-write semantics are implemented. Merge-on-read and distributed atomic commit certification are ongoing.</Callout>

<h2 id="syntax">Syntax</h2>

```sql
MERGE INTO <target_table> [AS <alias>]
USING <source_table_or_subquery> [AS <alias>]
ON <join_condition>
WHEN MATCHED [AND <condition>] THEN UPDATE SET <col> = <expr> [, ...]
WHEN MATCHED [AND <condition>] THEN DELETE
WHEN NOT MATCHED [AND <condition>] THEN INSERT (<cols>) VALUES (<exprs>)
```


<h2 id="example">Example</h2>

```sql
MERGE INTO inventory AS tgt
USING incoming_stock AS src
ON tgt.product_id = src.product_id
WHEN MATCHED AND src.quantity = 0 THEN DELETE
WHEN MATCHED THEN UPDATE SET tgt.quantity = tgt.quantity + src.quantity
WHEN NOT MATCHED THEN INSERT (product_id, quantity) VALUES (src.product_id, src.quantity)
```


<h2 id="return">Return</h2>
<p>The statement returns a single-row result with a `merged_rows` integer count.</p>

<h2 id="dml">Related DML</h2>
<p>For Iceberg tables, `DELETE FROM &lt;table&gt; WHERE &lt;predicate&gt;` and `UPDATE &lt;table&gt; SET &lt;col&gt; = &lt;expr&gt; WHERE &lt;predicate&gt;` are also intercepted and routed through copy-on-write Iceberg delete/update paths when the table is registered under a `KrishivCatalog`.</p>
