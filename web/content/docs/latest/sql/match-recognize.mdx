---
title: MATCH_RECOGNIZE
description: Complex Event Processing (CEP) pattern matching over event sequences.
status: Available
---
<h2 id="overview">Overview</h2>
<p>`MATCH_RECOGNIZE` detects sequences of rows that match a regular-expression-like pattern. Krishiv implements it via the built-in `PatternMatcher` and intercepts the statement before DataFusion parsing (DataFusion does not parse `MATCH_RECOGNIZE` natively).</p>

<h2 id="syntax">Syntax</h2>

```sql
SELECT *
FROM <source_table>
MATCH_RECOGNIZE (
  PARTITION BY <column> [, ...]
  ORDER BY <column> [ASC|DESC]
  MEASURES
    <pattern_var>.<column> AS <alias> [, ...]
  ONE ROW PER MATCH
  PATTERN (<pattern>)
  DEFINE
    <pattern_var> AS <condition> [, ...]
)
```


<h2 id="example">Example — Detect Price Rise followed by Drop</h2>

```sql
SELECT * FROM ticks
MATCH_RECOGNIZE (
  PARTITION BY symbol
  ORDER BY ts
  MEASURES
    FIRST(UP.price) AS start_price,
    LAST(DOWN.price) AS end_price
  ONE ROW PER MATCH
  PATTERN (UP+ DOWN+)
  DEFINE
    UP   AS price > PREV(price),
    DOWN AS price < PREV(price)
)
```


<h2 id="streaming-notes">Streaming Sources</h2>
<p>When the source table is an unbounded streaming source, Krishiv materialises a bounded window of recent events (default 100 000 rows) and runs the pattern matcher over that window. Set `KRISHIV_MATCH_RECOGNIZE_STREAMING_LIMIT` to override. Results cover only the collected window.</p>

<h2 id="clauses">Supported Clauses</h2>
<table class="api-table">
  <thead><tr><th>Clause</th><th>Support</th></tr></thead>
  <tbody>
    <tr><td>`PARTITION BY`</td><td>Available — matches run per partition key.</td></tr>
    <tr><td>`ORDER BY`</td><td>Available — rows within a partition are sorted before matching.</td></tr>
    <tr><td>`MEASURES`</td><td>Available — extract columns from matched pattern variables.</td></tr>
    <tr><td>`ONE ROW PER MATCH`</td><td>Available (default).</td></tr>
    <tr><td>`ALL ROWS PER MATCH`</td><td>Not yet supported.</td></tr>
    <tr><td>`PATTERN`</td><td>Available — quantifiers `+`, `*`, `?`, concatenation, alternation.</td></tr>
    <tr><td>`DEFINE`</td><td>Available — Boolean conditions referencing previous row values via `PREV()` and `FIRST()`/`LAST()`.</td></tr>
  </tbody>
</table>
