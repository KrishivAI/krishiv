<h2 id="constructors">Constructors</h2>
<table class="api-table">
  <thead><tr><th>Method</th><th>Description</th></tr></thead>
  <tbody>
    <tr><td>`Session.embedded()`</td><td>Create an in-process embedded session. No daemon or cluster needed.</td></tr>
    <tr><td>`Session.local()`</td><td>Create a single-node local session.</td></tr>
    <tr><td>`Session.from_env()`</td><td>Create a session using `KRISHIV_COORDINATOR` environment variable.</td></tr>
    <tr><td>`Session.connect(url: str)`</td><td>Connect to a remote Krishiv coordinator at `url`.</td></tr>
    <tr><td>`Session()`</td><td>Create a default embedded session (same as `Session.embedded()`).</td></tr>
  </tbody>
</table>

<Callout type="info" title="Common methods"> Most workloads use only a handful of `Session` methods. Start with: `embedded()`, `sql(query)`, `read_parquet(path)`, `register_parquet(name, path)`, `table(name)`, and `read_stream()`. The full tables below are the complete reference.</Callout>
