# Runtime Modes

Krishiv supports multiple runtime modes without splitting batch and streaming into separate engines.

| Mode | Use it for |
|---|---|
| Embedded | Tests, examples, and local API use. |
| Single-node | Local coordinator/executor validation on one host. |
| Distributed | Remote coordinator and executor workers. |
| Kubernetes | Operator, direct manifests, or Helm-managed deployments. |
