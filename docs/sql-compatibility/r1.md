# R1 SQL Compatibility

## Current Status

R1 executes local SQL through Apache DataFusion behind the `krishiv-sql`
crate. The public Krishiv API exposes Arrow `RecordBatch` results and keeps
DataFusion-specific types behind crate boundaries.

R1 supports embedded and single-node local execution with the same semantics.
The `distributed` mode is reserved for R2 and returns an explicit unsupported
feature error.

## Supported R1 SQL Subset

- `SELECT`
- projection
- `WHERE`
- simple aggregates and `GROUP BY`
- `LIMIT`
- local Parquet table registration
- `EXPLAIN` for logical and physical plans
- direct Parquet reads through the Rust API

## Supported Surfaces

- Rust API:
  - `Session::sql`
  - `Session::register_parquet`
  - `Session::read_parquet`
  - `DataFrame::collect`
  - `DataFrame::explain`
- CLI:
  - `krishiv sql --query <SQL>`
  - `krishiv sql --parquet <table=path> --query <SQL>`
  - `krishiv explain --query <SQL>`
  - `krishiv explain --parquet <table=path> --query <SQL>`
  - `krishiv jobs`

## Explicitly Out Of Scope For R1

- Certified join behavior.
- Window functions.
- Streaming SQL.
- DDL.
- DML.
- Lakehouse table writes.
- Spark SQL compatibility.
- Flink SQL compatibility.

## Compatibility Rule

Until the SQL compatibility matrix is stable, new SQL behavior must be
documented here as it is implemented. DataFusion may accept more SQL than this
file lists, but only the subset above is considered part of the R1 Krishiv
contract.
