# R1 SQL Compatibility

## Current Status

R1 bootstrap does not execute SQL yet. It only defines the command/API shape and a placeholder SQL planning seam.

## Planned R1 SQL Subset

The first real R1 SQL implementation should support:

- `SELECT`
- projection
- `WHERE`
- simple aggregates
- `LIMIT`
- local Parquet table registration
- `EXPLAIN` for logical and physical plans

## Explicitly Out Of Scope For R1 Bootstrap

- DataFusion-backed execution.
- Joins.
- Window functions.
- Streaming SQL.
- DDL.
- DML.
- Lakehouse table writes.
- Spark SQL compatibility.
- Flink SQL compatibility.

## Compatibility Rule

Until the SQL compatibility matrix is stable, new SQL behavior must be documented here as it is implemented.
